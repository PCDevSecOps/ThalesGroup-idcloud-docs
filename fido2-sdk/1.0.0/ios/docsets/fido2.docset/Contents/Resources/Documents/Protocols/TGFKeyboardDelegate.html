TGFKeyboardDelegate

@protocol TGFKeyboardDelegate <NSObject>

A delegate to receive relevant keyboard key events.

@since 1.0.0

  • -frameForKeyboard:

    The specific frame to set for the keyboard view. This callback serves as a input to generate a specific frame of the secure keyboard view. Frame initializations are final and MUST NOT be modified on runtime.

    Warning

    An exception is thrown if the frame is invalid for the current orientation. Rotation is currently unsupported.

    Declaration

    Objective-C

    - (CGRect)frameForKeyboard:(id<TGFKeyboard> _Nonnull)keyboard;

    Parameters

    keyboard

    An object conforming to the TGFKeyboard protocol.

    Return Value

    frame The intended keyboard frame to display on the UI.

  • -keyboard:didReceiveKey:

    This delegate method is invoked when a key is pressed on the keyboard view. The App layer should utilize this callback in order to determine the number of characters to be displayed on a secure input view.

    Key examples include: TGFKeyboardKeyInput - Add a new secure character (⬤) to the text field entry. TGFKeyboardKeyDelete - Remove a secure character from the text field entry. TGFKeyboardKeyDone - Passcode entry is submitted to the FIDO2 SDK. The App layer should await other

  • -keyboard:didFailAuthenticationWithRetryCountRemaining:

    A delegate callback invoked when the Passcode authentication fails with return of remaining retry count while the user still has attempt remaining. Retry count limit is 5 before TGFFido2ErrorCodeUserLockout error is triggered. Retry count will reset upon successful user authentication.

    Declaration

    Objective-C

    - (void)keyboard:(id<TGFKeyboard> _Nonnull)keyboard
    didFailAuthenticationWithRetryCountRemaining:
        (NSUInteger)retryCountRemaining;

    Parameters

    retryCountRemaining

    retry count remaining. On first authentication failed, the value of this property will be 4. On last authentication before it trigger TGFFido2ErrorCodeUserLockout, the value of this property will be 1.
  • -keyboard:didChangeUsage:

    A delegate callback invoked when the keyboard being displayed changed.
