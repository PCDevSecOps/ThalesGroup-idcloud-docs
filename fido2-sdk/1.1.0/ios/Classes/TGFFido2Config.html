TGFFido2Config

@interface TGFFido2Config : NSObject

A global FIDO2 configuration class. In general FIDO2 default behavior is to be as secure as possible. However, sometimes an application may wish to disable a specific security feature or even enhance a security feature. A small set of configuration options are exposed to the application to modify or add to FIDO2 default behavior.

@since 1.0.0

  • +setMaximumRetryCount:

    Sets the maximum number of possible retries before incurring a lockout penalty during a user verification process. This value defaults to 5. The minimum possible value is 1 otherwise the default value will be kept. The maximum possible value is 5 otherwise the default value will be kept. This value may only be configured ONCE per application runtime.

  • +setBaseLockoutDuration:

    Sets the base lockout penalty duration to be incurred upon exceeding the maximum number of possible retries during a user verification process. The penalty will increase exponentially with each subsequent verification failure. This value defaults to 30 seconds. The minimum possible value is 30 seconds otherwise the default value will be kept. The maximum possible value is 1800 seconds (i.e. 30 minutes) otherwise the default value will be kept. This value may only be configured ONCE per application runtime.
