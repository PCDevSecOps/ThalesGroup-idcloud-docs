  • -enroll

    Enroll a Thales Passcode Authenticator with a secure keyboard. Users MUST be enrolled to the specific authenticator to perform FIDO2 operations. The enrollment process stores a secure means of identification which will be used to verify the user when authentication is required.

    This authenticator will invoke the method passcodeAuthenticator:willStartKeyboard:operationInfo:cancelHandler on its delegate to provide a secure keyboard which should be displayed on the UI layer to identify the user.

    Note

    An error is delivered if this method is called on a previously enrolled user.

  • -unenroll

    Unenroll a previously enrolled Thales Passcode Authenticator. User authentication is required prior to a successful unenrollment. Upon successful unenrollment, all Thales Passcode FIDO2 registrations will be invalidated.

    This authenticator will invoke the method passcodeAuthenticator:willStartKeyboard:operationInfo:cancelHandler on its delegate to provide a secure keyboard which should be displayed on the UI layer to authenticate the user prior to unenrollment.

    Note

    An error is delivered if no user is enrolled to the Thales Passcode Authenticator.

  • -changePasscode

    Change the existing passcode for the current Thales Passcode Authenticator enrollment.

    This authenticator will invoke the method passcodeAuthenticator:willStartKeyboard:operationInfo:cancelHandler on its delegate to provide a secure keyboard which should be displayed on the UI layer to authenticate the user prior to changing the passcode.

    Upon authentication, the user would be requested to input and confirm a new passcode which SHALL then be used to for authentication henceforth.

  • +lockoutExpiryTimestamp:

    Retrieve a timestamp indicating the expiration of a lockout period. During the lockout period, authentication attempts will result in a TGFErrorUserLockout error. Per FIDO2 specification recommendations, FIDO2 SDK will start the lockout period at 30 seconds and exponentially increase after that, e.g. 30 seconds after 5th false attempt, 1 minute after 6th false attempt, etc. The lockout period will reset to 30 seconds upon a successful authentication.

    Note

    If there is no current lockout, this method will return 0.
